# 论文精读 Skill

## 概述

本 skill 指导 Claude 对学术论文进行深度精读，生成完整的中文精读报告。

**语言规范**：全程使用中文，但专业术语（token、embedding、attention、fine-tuning、backbone、loss、baseline 等）保留英文原文，不做翻译。

---

## Step 1：获取论文内容

### 输入类型 A：用户上传了 PDF 文件

1. 检查 `/mnt/user-data/uploads/` 目录，找到 PDF 文件路径。
2. 使用 `bash_tool` 提取全文：

```bash
pdftotext -layout /mnt/user-data/uploads/paper.pdf /tmp/paper.txt
cat /tmp/paper.txt
```

3. 若文字层为空（scanned PDF），改用视觉方式逐页 rasterize：

```bash
pdftoppm -jpeg -r 150 -f 1 -l 5 /mnt/user-data/uploads/paper.pdf /tmp/page
ls /tmp/page-*.jpg
```

然后用 `view` 工具读取图片进行视觉识别。

4. 对于含有复杂图表、架构图的页面，即使有文字层，也应 rasterize 关键页面以理解图示。

### 输入类型 B：用户提供论文 URL

常见来源及处理方式：

| 来源 | 策略 |
|------|------|
| arxiv.org/abs/XXXX | 将 URL 中 `/abs/` 改为 `/pdf/` 后 web_fetch，或直接 web_fetch abs 页面获取摘要+元数据，再搜索全文 |
| arxiv.org/pdf/XXXX | 直接 web_fetch |
| aclanthology.org | web_fetch 页面，找 PDF 链接后 fetch |
| openreview.net | web_fetch 页面获取 reviews + paper 内容 |
| 其他期刊/会议 | web_fetch 页面，提取摘要和可见文字；搜索是否有公开 PDF |

```javascript
// 示例：arXiv abs 页转 PDF
// https://arxiv.org/abs/2310.06825 → https://arxiv.org/pdf/2310.06825
```

若无法获取全文，至少获取：摘要、标题、作者、机构、发表年份，并在报告中注明"基于摘要及部分内容分析"。

---

## Step 2：快速定位论文结构

阅读全文前，先识别以下结构要素（用于后续精读导航）：

- **标题 / 作者 / 机构 / 发表年份 / 会议或期刊**
- **Abstract**：一句话提炼核心 claim
- **Introduction**：找 motivation、problem statement、contributions list
- **Related Work**：找对比的工作类别
- **Method / Model / Approach**：核心技术章节
- **Experiments**：数据集、指标、baseline、消融实验
- **Conclusion / Limitations**：作者自述的局限与未来方向

## 注意事项

### 深度要求
- **不做摘要复述**：不要把 Abstract 翻译一遍就结束。每个模块都要有自己的分析视角。
- **方法论要落地**：描述模型结构时，要说清楚"为什么这样设计"，不只是"是什么"。
- **数字要精确**：引用实验结果时，写出具体数值，不用"有所提升"这类模糊表达。
- **批判要有据**：提出疑问时，要基于论文内容，不要无端批评。

### 图表处理
- 若论文有架构图（Figure 1/2 通常是模型图），rasterize 该页并仔细分析图示关系。
- 实验结果表格：尽量还原关键数据到报告的 Markdown 表格中。

### 无法获取全文时
- 明确告知用户"仅基于摘要/部分内容"
- 尽力填写可知的模块，对未知部分注明 `（全文不可获取，此处基于摘要推断）`
- 建议用户提供 PDF 以获得完整分析

### 长论文处理
- 超过 30 页的论文（如技术报告），优先精读：Abstract、Introduction、核心方法章节、主实验表、Conclusion
- Appendix 中的内容按需查阅（消融实验、实现细节）
